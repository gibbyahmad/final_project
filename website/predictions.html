Implementation of Machine Learning using Random Forest


We chose to use random forest in order to predict the natural log of the sum of revenue from each Google Store user. We chose random forest because it uses both regression and classification through bootstrap aggregation, and combines multiple decision trees into one model, which renders predictions made with random forest more accurate on average. Each decision tree in our random forest considers a random subset of features and only has access to a random set of the data points in the training data set. Our prediction is therefore an average of all the individual decision tree estimates generated by the random forest. The random forest algorithm is ideal because it avoids the problem of overfitting, it can be used for both regression and classification, and it can be used for feature engineering (which was particularly critical for this project). Our model landed on an rmse of 1.058767 in relation to the natural log of the sum of all transactions per user. We used xgboost for purposes of comparison.
